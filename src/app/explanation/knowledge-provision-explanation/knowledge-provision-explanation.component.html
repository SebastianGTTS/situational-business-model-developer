Knowledge Provision

The first stage of Knowledge Provision is used by the Domain Experts to store their knowledge about methods into the Method Repository and the models into the Canvas Model Repository.

The Method Repository contains atomic Method Elements that are combined to Method Building Blocks, and, optionally, arranged to Method Patterns. The Method Elements are the tasks to accomplish, the phases to divide the method, the types to arrange the method, the stakholders that are involved, the artifacts that are created, the tools that are used, and the situational factors that should be covered. The Method Building Blocks combine a single task with multiple other elements to create an executable development step. The Method Patterns arrange types of elements in a predefined process to structure a development method.

The Canvas Model Repository contains atomic Canvas Elements that are hierachical structured though Canvas Building Blocks and visualized by Canvas Models. The Canvas Elements are the items of possible business decisions, the relationships for design support, the application domains to apply the knowledge, and the instances to group the knowledge. The Canvas Building Blocks structure the items as a hierchary together with relationship between and instances out of them. The Canvas Models visualize those building blocks together with guiding questions and examples.
